# 阶段2研究：核心数据功能

**阶段**: 2
**目标**: 实现比赛、赛事、球队、球员的核心数据展示和查询功能
**研究日期**: 2026-03-29

## 背景

阶段2需要实现五个核心详情页：比赛中心（首页）、比赛详情、赛事详情、球队详情、球员详情。这是一个较大的阶段，需要合理的任务拆分。

## Requirements 分析

### 比赛中心 (HOME-01 ~ HOME-07)
- 比赛列表展示（昨天/今天/明天）
- 日期筛选、联赛筛选、状态筛选
- 比赛卡片点击跳转
- 联赛名称跳转
- 热门资讯

### 比赛详情 (MATCH-01 ~ MATCH-09)
- 基本信息（联赛、轮次、时间、场地）
- 实时比分和状态
- 统计数据（控球率、射门、射正、角球等）
- 事件时间轴
- 首发/替补阵容
- 球员/球队/联赛跳转
- 相关资讯

### 赛事详情 (COMP-01 ~ COMP-08)
- 赛事基本信息
- 赛季切换
- 积分榜
- 赛程（按轮次筛选）
- 射手榜/助攻榜
- 球队列表
- 相关资讯

### 球队详情 (TEAM-01 ~ TEAM-09)
- 基本信息
- 阵容（按位置分类）
- 近期赛程
- 赛季数据统计
- 历史荣誉
- 联赛跳转
- 相关新闻

### 球员详情 (PLAYER-01 ~ PLAYER-07)
- 基本信息
- 赛季数据
- 生涯荣誉
- 能力维度图表
- 球队跳转
- 相关新闻

### 通用功能 (COMM-01)
- 双向跳转（比赛↔赛事↔球队↔球员↔资讯）

## 实施计划

### 阶段拆分策略

由于阶段2工作量较大，建议拆分为多个计划：

**Plan A: API 路由和数据层**
- 创建比赛、赛事、球队、球员的 API 路由
- 创建数据获取 Hooks
- 更新 mockData

**Plan B: 比赛中心（首页）增强**
- 完善筛选功能（联赛、状态）
- 优化比赛卡片
- 添加跳转链接

**Plan C: 比赛详情页**
- 创建比赛详情路由 `/match/[id]`
- 展示基本信息、比分、统计
- 事件时间轴
- 阵容展示

**Plan D: 赛事详情页**
- 创建赛事详情路由 `/competition/[id]`
- 积分榜、赛程、榜单
- 球队列表

**Plan E: 球队详情页**
- 创建球队详情路由 `/team/[id]`
- 阵容、赛程、数据、荣誉

**Plan F: 球员详情页**
- 创建球员详情路由 `/player/[id]`
- 能力维度图表（Recharts）
- 数据展示

### 路由设计

```
/                     - 比赛中心（首页）
/match/[id]          - 比赛详情
/competition/[id]    - 赛事详情
/team/[id]           - 球队详情
/player/[id]         - 球员详情
```

### 技术要点

1. **动态路由**: 使用 Next.js App Router 的 `[id]` 目录结构
2. **数据获取**: 使用 React Server Components 或客户端 Hooks
3. **图表**: 使用 Recharts 实现雷达图（球员能力）
4. **跳转**: 使用 Next.js Link 组件实现双向跳转
5. **类型**: 扩展 src/types/index.ts 添加更多类型

### 依赖关系

- Plan A (API/数据层) → 无依赖
- Plan B (比赛中心) → 依赖 Plan A
- Plan C (比赛详情) → 依赖 Plan A
- Plan D (赛事详情) → 依赖 Plan A
- Plan E (球队详情) → 依赖 Plan A
- Plan F (球员详情) → 依赖 Plan A
